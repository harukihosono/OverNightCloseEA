━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【時間帯の指定方法】
────────────────────────────────────────────────────

  形式: HHMM-HHMM

  例:
    2300-2359  → 23:00～23:59の間に決済
    0500-0600  → 5:00～6:00の間に決済
    1400-1500  → 14:00～15:00の間に決済
    2300-0100  → 23:00～翌1:00の間に決済 (日跨ぎOK)
    0000-0000  → 無効 (この曜日は決済しない)

  💡 この時刻は「日本時間」です！
     サーバー時刻への変換は自動で行われます。


【タイムゾーン設定】初心者はこれだけでOK！
────────────────────────────────────────────────────

  ■ TimezoneMode: JST (Auto DST) ← 推奨！
    → 日本時間で入力、夏時間/冬時間を自動判定
    → 初心者はこれを選べば間違いなし

  ■ SummerOffset: 6 (GMT+2サーバーの場合)
    → XM, FXGT など多くのブローカーはこの値でOK

  ■ WinterOffset: 7 (GMT+2サーバーの場合)
    → 同上

  サーバー時刻別の設定値:
    GMT+0サーバー → Summer: 9, Winter: 9
    GMT+2サーバー → Summer: 6, Winter: 7 ← 最も一般的
    GMT+3サーバー → Summer: 6, Winter: 6


【決済対象設定】
────────────────────────────────────────────────────

  ■ ClosePositions: true
    → 保有中のポジション(買い/売り)を決済
    → プロップファームでは必ずtrue！

  ■ ClosePendingOrders: true
    → 待機注文(Buy Limit, Sell Stop等)を削除
    → これも必ずtrue！(見落としがちなため)

  ■ MagicNumber: 0
    → 0 = 全てのポジション・注文を対象
    → 特定のEAだけ決済したい場合は、そのEAのマジックナンバーを入力


【決済後のアクション】
────────────────────────────────────────────────────

  ■ 決済のみ
    → 決済後もEAは稼働継続
    → デイトレードなど毎日使う場合に選択

  ■ 決済＋EA停止
    → 決済後にこのEAだけ停止
    → チャートからEAが削除される

  ■ 決済＋自動売買停止 ← プロップファームはこれ！
    → 決済後にMT4/MT5の自動売買ボタンを自動OFF
    → 全てのEAが停止され、新規ポジションが開かれない
    → 最も安全！

    ⚠️ この機能を使うには「DLLの使用を許可する」が必要


【通知設定】
────────────────────────────────────────────────────

  ■ EnableAlert: true
    → 決済時にアラート表示

  ■ EnableSound: true
    → 決済時にサウンド再生

  ■ AlertMinutesBefore: 5
    → 決済の5分前に事前通知
    → 「あと5分で決済します！」と教えてくれる
    → 0にすると事前通知なし


【決済設定】
────────────────────────────────────────────────────

  ■ MaxRetries: 3
    → 決済失敗時のリトライ回数
    → 通常は3でOK、不安な場合は5～10

  ■ RetryDelay: 1000
    → リトライ間の待機時間(ミリ秒)
    → 1000 = 1秒

  ■ Slippage: 3
    → 許容スリッページ(pips)
    → 市場が荒れている時は大きめに設定

❓ よくある質問 (FAQ)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q1: 本当に設定は簡単ですか？初心者でも大丈夫？
────────────────────────────────────────────────────
A: はい！たった3ステップ、3分で完了します。
   1. ファイルをコピー
   2. チャートにドラッグ&ドロップ
   3. 「Friday_CloseTime: 2300-2359」と入力
   これだけです。日本時間で入力するので分かりやすい！


Q2: サーバー時刻がGMT+2なのかGMT+3なのか分かりません
────────────────────────────────────────────────────
A: デフォルト設定(Summer:6, Winter:7)のままでOKです。
   ほとんどのブローカー(XM, FXGT, Exness等)はGMT+2です。

   確認方法:
   - MT4/MT5の「気配値表示」で現在のサーバー時刻を見る
   - 日本時間と比較する
   - 差が7時間ならGMT+2サーバーです


Q3: 手動で開いたポジションも決済されますか？
────────────────────────────────────────────────────
A: はい。MagicNumber=0の場合、手動ポジションも含めて
   全てのポジションが決済されます。

   特定のEAだけ決済したい場合は、MagicNumberを指定してください。


Q4: 複数通貨ペアのポジションがありますが大丈夫？
────────────────────────────────────────────────────
A: はい、全通貨ペアのポジションを一括決済します。
   通貨ペアごとに設定する必要はありません。

   ただし、EAは1つのチャートだけで稼働させてください。
   複数チャートで起動すると処理が重複します。


Q5: 決済に失敗したらどうなりますか？
────────────────────────────────────────────────────
A: リトライ機能があります。
   デフォルトで3回まで自動再試行します。

   それでも失敗した場合:
   - アラートとサウンドで通知
   - ログに詳細なエラー内容を出力
   - 残ったポジション数を表示


Q6: 毎日決済することは可能ですか？
────────────────────────────────────────────────────
A: はい！v2.0から可能になりました。
   全曜日に同じ時間帯を設定してください。

   例: 月～金曜まで全て「2300-2359」

   CloseActionは「決済のみ」を選択すれば、
   翌日も自動で稼働を継続します。


Q7: 週末以外でも使えますか？
────────────────────────────────────────────────────
A: もちろんです！
   - 毎日決済
   - 特定曜日だけ決済
   - 日跨ぎ決済(23:00-01:00)
   など、自由に設定できます。


Q8: DLL機能って何ですか？有効にしないとダメ？
────────────────────────────────────────────────────
A: DLLは「決済＋自動売買停止」機能で必要です。

   有効にする方法:
   1. MT4/MT5で「ツール」→「オプション」
   2. 「エキスパートアドバイザ」タブ
   3. 「DLLの使用を許可する」にチェック

   この機能を使わない場合は不要ですが、
   プロップファームでは有効化を強く推奨します。


Q9: バックテストできますか？
────────────────────────────────────────────────────
A: 時刻ベースの処理のため、バックテストは限定的です。
   ストラテジーテスターで動作確認は可能ですが、
   実際の決済動作はフォワードテストで確認してください。

   まずはデモ口座で1週間テストすることをお勧めします。


Q10: 夏時間・冬時間の切り替えは自動ですか？
────────────────────────────────────────────────────
A: はい！「JST (Auto DST)」モードなら完全自動です。

   米国夏時間(3月第2日曜～11月第1日曜)を自動判定し、
   サーバー時刻との時差を自動調整します。

   一度設定すれば、二度と触る必要はありません。

【問題1】決済されない
────────────────────────────────────────────────────
  ✓ チェックリスト:
    □ 自動売買ボタンがONになっているか？
    □ 曜日と時間帯の設定は正しいか？
    □ 「0000-0000」になっていないか？
    □ タイムゾーンモードは「JST (Auto DST)」か？
    □ EAがチャートに表示されているか？

  📋 ログ確認:
    「ターミナル」→「エキスパート」タブ
    → "Executing close for Friday" などのメッセージを確認
    → エラーメッセージがあれば内容を確認


【問題2】自動売買が停止されない
────────────────────────────────────────────────────
  ✓ チェックリスト:
    □ 「DLLの使用を許可する」にチェックが入っているか？
    □ CloseActionが「決済＋自動売買停止」になっているか？

  📋 確認方法:
    ツール → オプション → エキスパートアドバイザ
    → 「DLLの使用を許可する」を確認


【問題3】一部のポジションが残る
────────────────────────────────────────────────────
  ✓ チェックリスト:
    □ MagicNumberフィルターが設定されていないか？
    □ リトライ回数(MaxRetries)を増やしてみる
    □ スリッページ(Slippage)を大きくしてみる

  📋 ログ確認:
    エラーメッセージで失敗理由を確認
    → 「Invalid stops」→ スリッページを増やす
    → 「Timeout」→ リトライ回数を増やす


【問題4】時刻が合わない
────────────────────────────────────────────────────
  ✓ チェックリスト:
    □ TimezoneMode が「JST (Auto DST)」か？
    □ SummerOffset, WinterOffset は正しいか？
    □ サーバー時刻はGMT+何か確認したか？

  📋 確認方法:
    1. 気配値表示でサーバー時刻を見る
    2. 日本時間との差を計算
    3. 夏時間なら差-1時間、冬時間ならそのまま

    例: サーバー16:00、日本時間23:00
        → 差は7時間 → GMT+2サーバー
        → Summer:6, Winter:7 でOK

    起動時に "Filling mode: ORDER_FILLING_XXX" が表示されます

    - 通常は自動判定されるので問題なし
    - ログにエラーがあればブローカーに確認

【必ずお読みください】

  1. 【デモ口座でテスト】
     初めて使用する場合は、必ずデモ口座で1週間テストしてください。
     設定が正しいか、決済が正常に動作するか確認してから本番環境へ。

  2. 【プロップファームのルール確認】
     各プロップファームで週末持ち越しのルールは異なります。
     必ず利用しているプロップファームの規約を確認してください。

  3. 【市場クローズ時刻に注意】
     金曜日の市場クローズ時刻はブローカーによって異なります。
     決済時刻は市場クローズの30分～1時間前に設定してください。

     例: 市場が金曜24:00(土曜0:00)にクローズする場合
         → Friday_CloseTime: 2300-2359 (安全)

  4. 【1チャートのみ起動】
     このEAは1つのチャートでのみ起動してください。
     複数チャートで起動すると、決済処理が重複する可能性があります。

  5. 【VPS推奨】
     週末決済を確実に実行するため、VPS環境での運用を推奨します。
     自宅PCの場合、停電やネット切断で決済されないリスクがあります。

  6. 【事前通知を活用】
     AlertMinutesBeforeを5～10に設定し、決済前に確認できるようにしてください。
     万が一設定ミスがあっても、手動で対応できます。

  7. 【ログを定期確認】
     週に1度は「エキスパート」タブのログを確認してください。
     エラーや警告が出ていないか、正常動作しているかチェック。

  8. 【バージョンアップ】
     このREADMEを保管しておき、バージョンアップ時に参照してください。
